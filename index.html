What is Contenox?

Contenox is a self-hostable runtime for building and operating enterprise-grade AI agents, semantic search systems, and chat-driven applications. It models AI behaviour as explicit, observable state machines instead of opaque prompt chains.

Runtime, not just a library

Contenox is a complete runtime environment ‚Äì with API gateway, background workers, storage, and observability ‚Äì not just a collection of helper functions.

State machines, not black boxes

Workflows are explicit graphs of tasks, handlers, and transitions. Every step is visible, controllable, and replayable.

Chat-native interface

Users interact via chat, but under the hood each message drives a deterministic workflow ‚Äì from RAG queries to tool calls and side effects.

Vendor-agnostic & self-hosted

Run on-prem, in your cloud, or air-gapped. Route requests across OpenAI, Ollama, vLLM, or custom backends without locking yourself in.

Why it matters

Modern AI systems drift into opacity. Contenox brings logic, data flow, and side effects back into view ‚Äì so teams can reason about behaviour instead of guessing.

Sovereignty

You own the runtime, data, and decision boundaries. No hidden SaaS layer that can change behaviour without you noticing.

Reliability & control

Workflows are deterministic state machines: tasks, transitions, retries, and human approvals are defined explicitly instead of being buried in prompts.

Transparency & compliance

Every state change, model call, and hook invocation is logged and traceable ‚Äì a foundation for auditing, governance, and upcoming regulation.

AI workflows as explicit state machines

Contenox executes AI workflows as explicit state machines. Every step ‚Äì LLM call, data fetch, or external action ‚Äì is a task with defined inputs, outputs, and transitions.

Hooks: connect to anything

Extend agents with hooks ‚Äì remote services that integrate external APIs, databases, or tools. Register any OpenAPI-compatible endpoint and Contenox auto-generates callable tools for model execution.

Declarative & observable

Define workflows in YAML or via API. Each execution is logged and traceable: transitions, model calls, hook results, and intermediate state are all visible.

Knowledge-grounded workflows

Built-in RAG pipeline with document ingestion, embeddings, and vector search (e.g. via Vald). Agents answer based on your data, not just general training.

Multi-model orchestration

Route requests across Ollama, vLLM, OpenAI, or other providers. Configure fallbacks, affinity groups, and resolution policies on a per-task basis.

Architecture at a glance

Contenox is built as a modular, container-first system. The runtime can be embedded into existing stacks or deployed as its own AI infrastructure layer.

Core components

  • Backend: Go runtime & API gateway.
  • Workers: Python for parsing, embeddings, and long-running jobs.
  • Storage: PostgreSQL for metadata and state.
  • Cache & queues: Valkey / Redis.
  • Vector search: Vald for RAG indices.

Interface & deployment

  • Frontend: React + TypeScript chat client and visual builder.
  • Backend-for-frontend pattern for the UI.
  • Containerised and designed for Kubernetes.
  • Suitable for on-prem, cloud, or air-gapped setups.

Contenox is not a black-box SaaS. It is software you run and adapt ‚Äì a runtime that sits between LLMs and your real systems.
