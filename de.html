Lokales KI-Agent-CLI

Beschreibe ein Ziel.
Lass es erledigen.

KI-Workflows direkt auf der Kommandozeile.

Contenox wandelt natürliche Sprache in dauerhafte, schrittweise Pläne um und führt sie auf deinem Rechner mit echten Shell- und Dateisystem-Tools aus.
Keine Cloud. Kein Daemon. Nur ein einzelnes Binary.

$ contenox plan new "installiere einen git pre-commit Hook, der Commits blockiert, wenn go build fehlschlägt"
Erstelle Plan "installiere-einen-git-pre-commit-a3f9e12b" — 5 Schritte

$ contenox plan next --auto
Schritt 1: Notwendige Tools installieren... ✓
Schritt 2: .git/hooks/pre-commit erstellen... ✓
Schritt 3: Build-Check-Skript schreiben... ✓
Schritt 4: Bash-Inhalt in Hook-Datei schreiben... ✓
Schritt 5: chmod +x .git/hooks/pre-commit... ✓

# Das Modell hat diesen Hook geschrieben. Auf deinem Rechner.

Nicht noch ein Chatbot-Wrapper

Die meisten KI-CLI-Tools sind einmalig: eine Frage, eine Antwort, fertig. Contenox ist anders, weil es auf einem typisierten Ausführungs-Engine aufbaut – nicht auf einer Prompt-Schleife.

Andere Tools

Einmalig, zustandslos, vergessen

Copilot CLI schlägt einen Befehl vor. Cursor ergänzt Code. Mistral CLI trifft die Cloud-API. Keines davon erinnert sich, was passiert ist, kann einen fehlgeschlagenen Schritt wiederholen oder eigenständig einen 5-Schritte-Plan ausführen, während du dir einen Kaffee holst.

Contenox

Dauerhafte Pläne, echte Ausführung

Pläne überstehen Neustarts. Jedes Schritt-Ergebnis wird in SQLite gespeichert. Du kannst mitten im Plan pausieren, prüfen was passiert ist, einen bestimmten Schritt wiederholen oder das Modell den aktuellen Stand neu planen lassen. Es ist ein Workflow-Engine, den du vom Terminal aus bedienst.

Echter Ausführungs-Engine unter der Haube

Contenox führt eine typisierte Zustandsmaschine aus: Verzweigungslogik, Retry-Richtlinien, Multi-Modell-Routing, Tool-Call-Dispatch, Token-Budget-Management. Die CLI-Oberfläche ist einfach. Was darunter steckt, nicht.

Human-in-the-loop als Standard

contenox plan next führt einen Schritt aus und hält an. Du prüfst. Du entscheidest. --auto schaltet volle Autonomie frei — nur wenn du es sagst. Das ist kein nachgereihtes Sicherheitsfeature. Es ist das Design.

Deine Chains, deine Modelle

Workflows sind JSON-Dateien, die dir gehören. Tausche Modelle pro Task aus. Füge externe Hooks für alles hinzu, das eine HTTP-API hat. Läuft vollständig offline mit Ollama. Oder nutze OpenAI, Gemini oder einen OpenAI-kompatiblen Endpunkt.

Keine Infrastruktur

Einzelnes Binary. SQLite. Kein Daemon, kein Docker, kein NATS, kein Postgres. Leg es auf einem beliebigen Rechner ab und es funktioniert. Der Runtime-API-Server ist für Team- und Produktionseinsatz verfügbar — aber das CLI braucht nichts davon.

Installieren

Ein Befehl. Binary verschieben. Fertig.

# Neueste Version herunterladen
TAG=$(curl -sL https://api.github.com/repos/contenox/contenox/releases/latest | grep '"tag_name"' | cut -d'"' -f4)
ARCH=$(uname -m | sed 's/x86_64/amd64/;s/aarch64/arm64/')
curl -sL "https://github.com/contenox/contenox/releases/download/${TAG}/contenox-${TAG}-linux-${ARCH}" -o contenox
chmod +x contenox && sudo mv contenox /usr/local/bin/contenox
$ contenox --version

# Lokales Modell
ollama serve && ollama pull qwen2.5:7b

Cloud-Anbieter (OpenAI, Gemini, vLLM) funktionieren ebenfalls — API-Schlüssel in .contenox/config.yaml nach contenox init eintragen.

Wie es funktioniert

Drei Modi. Ein Binary.

contenox plan — Autonome Aufgabenausführung

Beschreibe ein Ziel. Das LLM generiert einen Plan. Schritte werden in SQLite gespeichert.
Führe einen Schritt zur Zeit mit plan next aus, oder gehe voll automatisch mit plan next --auto. Wiederhole, überspringe oder plane jeden Schritt neu.

contenox — Interaktiver Chat

Natürliche Sprache → Shell-Befehle → Antwort. Chat-Verlauf bleibt über Sitzungen hinweg erhalten. Nutze deine eigene Chain mit --chain oder die Standardchain.

contenox exec — Skriptbare Chains

Führe beliebige Chains mit beliebigen Eingabetypen zustandslos aus. Pipe-freundlich. Perfekt für CI-Skripting, Batch-Jobs oder das Verbinden von Chains.

Bring deinen eigenen Workflow

Chains sind JSON. Definiere Tasks, Handler, Verzweigungsbedingungen, Tool-Hooks und Modell-Konfigurationen. Tausche Modelle pro Schritt aus. Füge jeden HTTP-Endpunkt als Tool hinzu. Der Engine erledigt den Rest.

Bereit zum Ausprobieren?

Ein Binary. Funktioniert auf deinem Laptop, deinem Server oder air-gapped.
